01 Numeric Data Types

Integer Data Types

Integer types store whole numbers without decimal points. Choosing the right integer type saves storage and improves performance.

Data Type Storage Range When to Use
TINYINT 1 byte 0 to 255 (unsigned only) Flags, status codes, ratings, very small value cases.
SMALLINT 2 bytes −32,768 to 32,767 When a large range is not needed. Saves space for smaller integer values.
INT 4 bytes −2,147,483,648 to 2,147,483,647 (±2 billion) Most commonly used integer type. Used for IDs, counts, general numeric values.
BIGINT 8 bytes −9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 Huge datasets, large transaction IDs, logs.

DECIMAL / NUMERIC

Both DECIMAL and NUMERIC are used to store exact numeric values (unlike FLOAT, which can lose precision). They are identical for most databases.

DECIMAL(p, s) / NUMERIC(p, s) p = precision → total number of digits s = scale → digits after the decimal point digits before decimal = (p − s)
💡
Syntax Example

price DECIMAL(5, 2) — valid values: 123.45, 9.99

✨
When to Use DECIMAL / NUMERIC

Money & financial data · Percentages · Calculations where rounding errors are unacceptable.

FLOAT / REAL

FLOAT and REAL are approximate numeric data types used to store floating-point numbers. They do not store exact values — they store approximations.

age_ratio FLOAT age_ratio FLOAT(n) -- n = number of bits for precision (optional) FLOAT(1–24) → behaves like REAL (4 bytes) FLOAT(25–53) → high precision (8 bytes)
Feature DECIMAL / NUMERIC FLOAT / REAL
Type Exact Approximate
Precision Exact (user-defined) Approximate
Storage Based on precision Fixed (4 or 8 bytes)
Accuracy 100% accurate May have rounding errors
Use Cases Money, finance Scientific, large calculations
Example 0.1 + 0.3 = 0.40 ✔ 0.1 + 0.3 ≠ 0.40 ✘

Numeric Functions & Operations

Arithmetic Operators

Standard arithmetic operators: + (add)  - (subtract)  * (multiply)  / (quotient)  % (remainder).

SQL 
SELECT 10/3  AS quotient,
       10%3  AS remainder;
-- Result: Quotient = 3,  Remainder = 1
⚡
Integer Division Gotcha

SELECT 10/3 returns 3 (integer), not 3.333…. To get the exact decimal result, cast the numerator as FLOAT or write 10.0/3.

Rounding: ROUND, CEILING, FLOOR

ROUND(number, decimal_places)

Rounds a number to the specified number of decimal places. Negative values round to the left of the decimal point.

SQL 
SELECT ROUND(123.4567,  2) AS RoundedValue;  -- 123.46
SELECT ROUND(123.4567,  0) AS RoundedValue;  -- 123
SELECT ROUND(123.4567, -1) AS RoundedValue;  -- 120  (rounds to tens)
ℹ️
Fact

Negative decimal places round to the left of the decimal point (tens, hundreds, etc.).

CEILING(number)

Rounds a number up to the nearest integer, regardless of the decimal fraction. Always moves toward positive infinity.

SQL 
SELECT CEILING( 123.456) AS CeilValue;  --  124
SELECT CEILING(-123.456) AS CeilValue;  -- -123

FLOOR(number)

Rounds a number down to the nearest integer, regardless of the decimal fraction. Always moves toward negative infinity.

SQL 
SELECT FLOOR( 123.456) AS FloorValue;  --  123
SELECT FLOOR(-123.456) AS FloorValue;  -- -124
Function Rounds To Positive Example Negative Example Typical Use Case
ROUND Nearest value 123.456 → 123.46 −123.456 → −123.46 Financial rounding, reports
CEILING Up (toward +∞) 123.456 → 124 −123.456 → −123 Estimating minimum full units needed
FLOOR Down (toward −∞) 123.456 → 123 −123.456 → −124 Truncating, limits, minimum thresholds

02 Character / String Data Types

String Data Types Overview

CHAR(n)

Fixed-length non-Unicode characters. Always uses exactly n characters. Pads with spaces if shorter.

VARCHAR(n)

Variable-length non-Unicode characters. Stores only what you insert — no padding.

TEXT

Used for storing very large non-Unicode text. Deprecated — prefer VARCHAR(MAX).

NCHAR(n)

Same as CHAR but stores Unicode characters (2 bytes per char). Supports Bangla, Arabic, etc.

NVARCHAR(n)

Unicode version of VARCHAR. The most common choice for multilingual applications.

Unicode vs Non-Unicode

Type Supports Unicode Example Use
CHAR ❌ No English only
VARCHAR ❌ No English only
NCHAR ✅ Yes Bangla, Arabic, CJK
NVARCHAR ✅ Yes Multilingual apps

Full Comparison

Unicode ❌ No ❌ No ❌ No ✅ Yes ✅ Yes

Performance Fast (fixed) Flexible Slower Fast Flexible

📌
What is Padding?

Padding means adding extra spaces to fill the defined length. Example: DECLARE @x CHAR(10) = 'Hi' — what you expect: 'Hi' — what is actually stored: 'Hi        ' (8 trailing spaces added to reach length 10).

String Functions

Function Purpose Returns Key Behavior Common Use Case Important Notes
LEN Count characters Integer (length) Ignores trailing spaces (SQL Server) Validate string size, filtering Does NOT count trailing spaces
LEFT Extract from start String Takes first n characters from left Get prefix (e.g., first name initial) If n > length → returns full string
RIGHT Extract from end String Takes last n characters from right Get suffix (e.g., last digits of phone) Works opposite of LEFT()
SUBSTRING Extract from any position String Starts at position and takes length chars Flexible extraction anywhere Position starts at 1 (not 0)
LEN(column_name) LEFT(column_name, n) RIGHT(column_name, n) SUBSTRING(column_name, start_position, length) -- To make dynamic: length = LEN(column)
💎
LEN() vs DATALENGTH()

LEN() ignores trailing spaces but counts spaces at the beginning. DATALENGTH() counts actual bytes including all spaces. SUBSTRING is more powerful than LEFT and RIGHT combined.

TRIM, LTRIM, RTRIM

All three functions remove unwanted spaces (or specified characters) from text.

TRIM() LTRIM() RTRIM()
✨
Special Case — Remove Specific Characters

SELECT TRIM('#' FROM '###Data###') → Result: 'Data'

Case Conversion

UPPER() LOWER()

UPPER() converts all letters to uppercase. LOWER() converts all letters to lowercase.

REPLACE

REPLACE(original_string, old_substring, new_substring)

Replaces all occurrences of old_substring in original_string with new_substring.

🔧
Use Case 1 — Remove Unwanted Characters

Converts 017-123-4567 → 0171234567

SQL 
-- Remove dashes from phone numbers
SELECT REPLACE(phone_number, '-', '') AS clean_number
FROM customers;

-- Replace multiple words (nested REPLACE)
SELECT REPLACE(REPLACE(description, 'bad', 'good'), 'poor', 'excellent')
FROM reviews;

-- Mask sensitive data (credit card)
SELECT REPLACE(card_number, LEFT(card_number, 12), '************')
FROM payments;
-- 1234567890123456 → ************3456

Pattern Matching: LIKE, %, _, PATINDEX

Used to search for a pattern inside a column.

Wildcard Meaning Example
% Any number of characters (0, 1, or many) LIKE 'A%' → starts with A
_ Exactly one character LIKE '_at' → cat, bat, hat

PATINDEX()

PATINDEX is a function (not an operator like LIKE). It returns the position number of the first matching pattern. Returns 0 if no match is found.

SQL 
SELECT PATINDEX('%son%', 'Johnson') AS position;
-- Result: 5  (pattern 'son' found at position 5 in 'Johnson')

FORMAT Function

FORMAT(value, format [, culture])

The FORMAT function returns a value formatted with the specified format string and optional culture. Primarily used for formatting dates, times, and numbers into human-readable strings.

✨
Parameters

value — the value to format  ·  'format' — format string (standard or custom)  ·  culture — optional locale (e.g., 'en-US', 'fr-FR')

Custom Number Formatting

SQL 
SELECT FORMAT(1234567, '##,##,##,###');  -- 12,34,567

Date Formatting

SQL 
SELECT
    FORMAT(GETDATE(), 'yyyy-MM-dd')    AS ISO_Format,
    FORMAT(GETDATE(), 'dd/MM/yyyy')    AS British_Format,
    FORMAT(GETDATE(), 'MMMM dd, yyyy') AS Full_Format,
    FORMAT(GETDATE(), 'ddd')           AS Day_Short,
    FORMAT(GETDATE(), 'dddd')          AS Day_Full;

Number Formatting (Currency, Percent)

SQL 
SELECT
    FORMAT(1234.56, 'C') AS Currency,    -- $1,234.56
    FORMAT(1234.56, 'N') AS Number,      -- 1,234.56
    FORMAT(0.756,   'P') AS Percentage;  -- 75.60%
-- Output depends on culture setting

Culture / Localization

SQL 
SELECT
    FORMAT(1234.56, 'C', 'en-US') AS US_Format,     -- $1,234.56
    FORMAT(1234.56, 'C', 'fr-FR') AS French_Format; -- 1 234,56 €
-- Very useful for international reports

FORMAT() for Custom Patterns

📝 Task

Convert date to compact format: 2026-04-20 → 20260420

SQL 
SELECT FORMAT(GETDATE(), 'yyyyMMdd') AS Compact_Date;
-- Output: 20260409

Time Formatting

📝 Task

Format to show time only (ignore the date)

SQL 
SELECT FORMAT(GETDATE(), 'hh:mm tt') AS Time_12hr;
-- Output: 06:45 PM

FORMAT vs CONVERT — Comparison

Feature FORMAT() CONVERT()
Flexibility Very High Limited
Performance Slow Fast
Culture Support Yes No
Output Type NVARCHAR Depends on style
Use Case Reporting / display Data processing
⚡
Best Practice Note

Think of FORMAT() as a presentation layer tool, not a data-processing tool. Use it when preparing final reports needing custom display formats. Avoid it with large datasets, aggregations, or joins — it performs poorly at scale.
