01 What is NULL?

NULL represents the absence of a value — it means the data is unknown, missing, or not applicable. It is not a data category itself.

⚠️
Critical

NULL is not the same as zero (0), an empty string (''), or false. It means "we don't know what the value is."

⚠️
Common Mistake

You cannot use = NULL or <> NULL to find or exclude NULLs. This always returns no results. Always use IS NULL or IS NOT NULL.

NULL leads to misleading outcomes if not handled properly — especially in comparisons, calculations, and JOIN conditions.

02 NULL Behavior

Row Level (Arithmetic)

Adding or concatenating any value with a NULL cell returns NULL. Example: 100 + NULL = NULL

Column Level (Aggregations)

Functions like SUM(), AVG(), COUNT(col) silently ignore NULL rows. Only COUNT(*) counts them.

ORDER BY — Sorting

ASC: NULLs appear at the top (treated as lowest).
DESC: NULLs appear at the bottom (treated as lowest).

Comparisons

Any comparison with NULL (=, <, >) returns UNKNOWN, not TRUE or FALSE. This is the "three-valued logic" of SQL.

ℹ️
Three-Valued Logic

SQL uses TRUE, FALSE, and UNKNOWN. A WHERE clause only returns rows where the condition is TRUE — rows evaluating to UNKNOWN (due to NULL) are excluded.

NULL in JOINs

When a JOIN condition involves a column containing NULLs, those rows are excluded from the result — because NULL = NULL evaluates to UNKNOWN, not TRUE.

SQL 
-- NULL in join key = row excluded from output
SELECT * FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
-- Any row where customer_id is NULL on either side is dropped

NULL in GROUP BY

GROUP BY treats all NULLs as belonging to the same group. So multiple NULL values in a grouping column are grouped together into one row.

03 NULL Functions Overview

There are three fundamental operations when dealing with NULLs:

04 IS NULL / IS NOT NULL — Identify

Used in a WHERE clause to filter records based on the presence or absence of NULL values.

Syntax

column_name IS NULL
column_name IS NOT NULL

IS NULL — Find Missing Data

SQL 
-- Find all customers who have not provided an email
SELECT CustomerID, CustomerName
FROM sales.Customers
WHERE Email IS NULL;

IS NOT NULL — Find Present Data

SQL 
-- Find all customers who have provided a phone number
SELECT CustomerID, CustomerName, Phone
FROM sales.Customers
WHERE Phone IS NOT NULL;

Finding Customers with No Orders (LEFT JOIN + IS NULL)

SQL 
-- List all customers who have never placed an order
SELECT C.*
FROM sales.Customers AS C
LEFT JOIN sales.Orders AS O
  ON C.CustomerID = O.CustomerID
WHERE O.CustomerID IS NULL;
-- LEFT JOIN keeps all customers; WHERE IS NULL filters to those with no match
💡
Tip

The pattern LEFT JOIN + WHERE … IS NULL is one of the most common and useful techniques in SQL to find records that don't exist in another table.

05 ISNULL — Fill / Replace NULL

ISNULL is a SQL Server function that replaces a NULL value with a meaningful alternative.

Syntax

ISNULL( column_to_check, replacement_value )

How it Works

  • If column_to_check is NULL → returns replacement_value
  • If column_to_check is not NULL → returns the original value unchanged
  • Exactly 2 arguments, no more, no less

Example — Replace NULL address

SQL 
-- Replace NULL order address with shipping address
SELECT
    OrderID,
    ISNULL(OrderAddress, ShippingAddress) AS FinalAddress
FROM sales.Orders;

Example — ISNULL in JOIN conditions

SQL 
-- Prevent NULL-caused row exclusion in a JOIN
SELECT *
FROM sales.OrdersA AS a
JOIN sales.OrdersB AS b
  ON ISNULL(a.Type, '') = ISNULL(b.Type, '');
-- NULLs in Type are treated as '' (empty string), so they match each other

Limitations

  • Limited to exactly 2 arguments only
  • SQL Server specific — Oracle uses NVL(), MySQL uses IFNULL()
  • The replacement value must be compatible in data type with the original column

06 COALESCE — Fill / Replace (Multi-Value)

COALESCE is a standard SQL function that evaluates a list of expressions and returns the first non-NULL value it finds.

Syntax

COALESCE( expr1, expr2, expr3, … )

How it Works

  • Checks expressions left to right
  • Returns the first non-NULL value found
  • If all expressions are NULL → returns NULL
  • Accepts unlimited arguments

Example — Priority contact method

SQL 
-- Return the first available contact: mobile → email → phone → 'No Contact'
SELECT
    CustomerID,
    COALESCE(MobileNumber, Email, PhoneNumber, 'No Contact Available') AS PrimaryContact
FROM sales.Customers;

Example — Fallback display name

SQL 
-- Use nickname if available, else full name, else 'Unknown'
SELECT
    EmployeeID,
    COALESCE(Nickname, FirstName + ' ' + LastName, 'Unknown') AS DisplayName
FROM HR.Employees;

Example — Null-safe JOIN

SQL 
-- Same as ISNULL example but using COALESCE (cross-platform)
SELECT *
FROM sales.OrdersA AS a
JOIN sales.OrdersB AS b
  ON COALESCE(a.Type, '') = COALESCE(b.Type, '');

Example — Custom NULL sort (NULLs last)

SQL 
-- Sort customers low-to-high score; NULL scores appear LAST
SELECT CustomerID, Score
FROM sales.Customers
ORDER BY
    CASE WHEN Score IS NULL THEN 1 ELSE 0 END,  -- push NULLs to bottom
    Score ASC;                                     -- then sort normally
💡
Pro Tip

Always end your COALESCE list with a hard-coded string like 'N/A' or 'Unknown' as the final fallback. This guarantees the column will never display NULL to end users.

07 ISNULL vs COALESCE — Full Comparison

CriteriaISNULLCOALESCE
Arguments Exactly 2 only 2 or more (unlimited)
Performance Faster — single evaluation Slower — evaluates each arg in order
SQL Standard Non-standard — SQL Server only ANSI Standard — all major DBs
Equivalent in Oracle NVL(col, val) COALESCE(col, val1, …)
Equivalent in MySQL IFNULL(col, val) COALESCE(col, val1, …)
Data Type Resolution Uses the data type of the first argument Uses the highest precedence data type among all args
Best Used When Simple single replacement in SQL Server for performance Multiple fallback values, cross-platform, or complex logic
Returns NULL if all NULL? Yes, if replacement is also NULL Yes, if all expressions are NULL
ℹ️
Recommendation

Prefer COALESCE in most scenarios — it is portable, flexible, and expressly designed for multi-value NULL handling. Use ISNULL only in SQL Server when you need the small performance edge in a tight query.

08 NULLIF — Create NULL

NULLIF returns NULL if two expressions are equal; otherwise it returns the first expression.

Syntax

NULLIF( expression1, expression2 )

How it Works

  • If expression1 = expression2 → returns NULL
  • If they are different → returns expression1 unchanged
  • Equivalent to: CASE WHEN expr1 = expr2 THEN NULL ELSE expr1 END

Key Use Case — Avoiding Division by Zero

SQL 
-- Calculate sales price per unit (avoid division by zero)
SELECT
    OrderID,
    Sales,
    Quantity,
    Sales / NULLIF(Quantity, 0) AS SalesPrice
FROM sales.Orders;
-- If Quantity = 0, NULLIF returns NULL → NULL / anything = NULL (no error)
⚠️
Without NULLIF

Dividing by zero in SQL Server throws a runtime error: "Divide by zero error encountered." Always protect division operations with NULLIF(denominator, 0).

Use Case — Replace Sentinel Values with NULL

SQL 
-- Old system stored 'N/A' as a placeholder string
-- Convert 'N/A' → NULL for clean data
SELECT
    CustomerID,
    NULLIF(Region, 'N/A') AS CleanRegion
FROM sales.Customers;

Combining NULLIF + COALESCE

SQL 
-- Convert 0 to NULL, then replace NULL with a default of 1
SELECT
    OrderID,
    COALESCE(NULLIF(Quantity, 0), 1) AS SafeQuantity
FROM sales.Orders;

09 NULL Function Decision Tree

Use this tree to decide which NULL function to reach for:

What do you want to do with NULL?
├── Find or filter rows that have NULL
→ Use IS NULL or IS NOT NULL in WHERE clause
├── Replace NULL with a default value
How many fallback values do you need?
├── Only one fallback → ISNULL(col, default)
└── Two or more fallbacks → COALESCE(col, fallback1, fallback2, …)
└── Turn a specific value INTO NULL
→ Use NULLIF(expression, value_to_nullify)
-- Especially useful to prevent division by zero

10 ⚡ Notable Facts About NULL

🔢

NULL ≠ 0 ≠ ''
NULL means unknown. Zero means "I know the value is nothing." Empty string means "I know there's no text here."

🧮

NULL arithmetic always gives NULL
NULL + 5, NULL * 100, 'text' + NULL — all return NULL.

❌

NULL = NULL is UNKNOWN
Two NULLs are not equal to each other in SQL. You cannot check equality with =.

📊

COUNT(*) vs COUNT(col)
COUNT(*) counts all rows including NULLs. COUNT(col) skips NULL values.

🔗

NULL in UNIQUE constraints
Most databases (including SQL Server) allow multiple NULLs in a UNIQUE column because NULL ≠ NULL.

🔄

NULL propagates in CASE
If no WHEN matches and there's no ELSE, CASE returns NULL by default.

📋

GROUP BY groups NULLs together
Multiple NULL values in a GROUP BY column are treated as one group, not separate rows.

🏆

AVG ignores NULLs silently
AVG() divides by the count of non-NULL values only — this can skew results significantly.

📌

NOT IN with NULLs is dangerous
col NOT IN (1, 2, NULL) returns no rows. Always use NOT EXISTS instead when NULLs may be present.
