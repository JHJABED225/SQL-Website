01 Introduction to SQL Server

What is SQL?

SQL stands for Structured Query Language. It is the standard language used to interact with relational databases.

Store Data Retrieve Data Filter Data Modify Data Delete Data Control Structures Analyze Business Info

What is SQL Server?

SQL Server is a Relational Database Management System (RDBMS) developed by Microsoft Corporation. It uses T-SQL (Transact-SQL) as its primary query language.

03 SQL Query Processing Basics

Logical Query Processing Order

Even though we write queries in one order, SQL Server processes them logically in a different sequence. Understanding this is critical for writing correct and optimized queries.

Execution OrderClauseWhat Happens
1FROMIdentifies the data source / joins tables
2WHEREFilters individual rows
3GROUP BYGroups filtered rows
4HAVINGFilters groups
5SELECTSelects and computes columns
6ORDER BYSorts the final result set
💎
Why This Matters
Understanding logical processing order is key for writing optimized queries, debugging errors (like "column alias not found"), writing analytical reports, and acing SQL interview questions. Column aliases defined in SELECT cannot be used in WHERE because SELECT runs after WHERE.

Basic Syntax

Selecting Specific Columns

FROM Employees;

Selecting All Columns

T-SQL

FROM Employees;
⛔
Avoid SELECT * in Production
Using SELECT * returns unnecessary columns, increases network usage, slows reporting systems, reduces readability, and may expose sensitive data. Always select only the columns you actually need.

    Salary    AS MonthlySalary
FROM Employees;

T-SQL

    FirstName,
    Salary,
    Salary * 12 AS AnnualSalary,
    Salary * 0.10 AS Bonus,
    Salary + 5000 AS RevisedSalary
FROM Employees;

String Concatenation

T-SQL
-- Basic concatenation
SELECT
    FirstName + ' ' + LastName AS FullName
FROM Employees;

-- NULL-safe concatenation using ISNULL
SELECT
    FirstName + ' ' + ISNULL(LastName, '') AS FullName
FROM Employees;
⚠️
NULL Breaks Concatenation
If LastName is NULL, the entire concatenated result becomes NULL. Always use ISNULL() or COALESCE() when concatenating nullable columns.

05 FROM Clause

The FROM clause specifies the data source — the table, view, subquery, or join from which data is retrieved.

Basic Usage & Schema References

T-SQL
-- Simple table reference
SELECT * FROM Employees;

-- Schema-qualified reference (recommended)
SELECT * FROM sales.employees;

-- Table alias shorthand
SELECT e.FirstName, e.Salary
FROM sales.employees e;

Multiple Tables with JOIN

T-SQL
SELECT
    c.CustomerName,
    o.TotalAmount
FROM Customers c
INNER JOIN Orders o
    ON c.CustomerID = o.CustomerID;

Derived Tables (Subqueries in FROM)

T-SQL
SELECT *
FROM
(
    SELECT
        Department,
        AVG(Salary) AS AvgSalary
    FROM Employees
    GROUP BY Department
) AS DepartmentSummary;

Best Practices

PracticeBenefit

06 WHERE Clause

The WHERE clause filters rows before they are returned. It is processed after FROM but before GROUP BY.

Comparison Operators

OperatorMeaning
=Equal to
!= or <>Not equal to
>Greater than
<Less than
>=Greater than or equal
<=Less than or equal

Logical Operators & Filtering Examples

T-SQL
-- AND: both conditions must be true
SELECT * FROM Employees
WHERE Department = 'HR' AND Salary > 40000;

-- OR: either condition true
SELECT * FROM Employees
WHERE Department = 'HR' OR Department = 'Finance';

-- NOT: negates condition
SELECT * FROM Employees
WHERE NOT Department = 'HR';

-- BETWEEN: inclusive range
SELECT * FROM Employees
WHERE Salary BETWEEN 40000 AND 70000;

-- IN: match any value in list
SELECT * FROM Employees
WHERE Department IN ('HR', 'Finance', 'Sales');

-- EXISTS: check for related rows
SELECT * FROM Customers c
WHERE EXISTS
(
    SELECT 1 FROM Orders o
    WHERE o.CustomerID = c.CustomerID
);

NULL Filtering — Critical Rule

T-SQL
-- ❌ WRONG: = NULL never returns TRUE
WHERE ManagerID = NULL

-- ✅ CORRECT: use IS NULL / IS NOT NULL
WHERE ManagerID IS NULL
WHERE Email     IS NOT NULL

Operator Precedence — Watch Out!

⛔
AND Executes Before OR
Without parentheses, AND binds tighter than OR. The query below does not behave as intended — it applies the salary filter only to Finance, not HR.
T-SQL
-- ❌ Dangerous: AND binds before OR
SELECT * FROM Employees
WHERE Department = 'HR'
   OR Department = 'Finance'
  AND Salary > 50000;

-- ✅ Correct: use parentheses
SELECT * FROM Employees
WHERE
(
    Department = 'HR'
    OR Department = 'Finance'
)
AND Salary > 50000;

Common Mistakes

MistakeResult
Missing WHERE in UPDATE/DELETEMassive unintended data loss
Using = NULLNo rows returned (silent failure)
Ignoring AND/OR precedenceWrong, hard-to-detect output

07 ORDER BY Clause

The ORDER BY clause sorts the final result set. It is the last clause processed logically, which means it can reference aliases defined in SELECT.

T-SQL
-- Ascending (default)
SELECT * FROM Employees ORDER BY Salary ASC;

-- Descending
SELECT * FROM Employees ORDER BY Salary DESC;

-- Multiple columns
SELECT * FROM Employees
ORDER BY Department ASC, Salary DESC;

-- Sort by alias
SELECT Salary * 12 AS AnnualSalary
FROM Employees
ORDER BY AnnualSalary DESC;

-- Recent orders first
SELECT * FROM Orders ORDER BY OrderDate DESC;
💡
NULL Sorting Behavior
In SQL Server, NULL values appear first when sorting in ascending order. Keep this in mind when generating reports where nulls should appear last.
⚠️
Sorting Has a Cost
Sorting is expensive — especially on large datasets, multiple columns, or non-indexed columns. Create indexes on frequently sorted columns to improve performance.

08 GROUP BY Clause

The GROUP BY clause groups rows that share the same values in specified columns so that aggregate functions can be applied to each group.

Aggregate Functions

FunctionPurpose
SUM(col)Total of all values in the group
AVG(col)Average of all values
MIN(col)Minimum value
MAX(col)Maximum value
COUNT(*)Count of all rows (including NULLs)
COUNT(col)Count of non-NULL values only
T-SQL
-- Average salary per department
SELECT
    Department,
    AVG(Salary) AS AverageSalary
FROM Employees
GROUP BY Department;

-- Multiple column grouping
SELECT
    Department,
    Status,
    COUNT(*) AS EmployeeCount
FROM Employees
GROUP BY Department, Status;
⛔
Grouping Rule — Common Error
Every non-aggregated column in SELECT must appear in GROUP BY. The query below will throw an error because Salary is neither aggregated nor grouped.
SELECT Department, Salary FROM Employees GROUP BY Department; ← Error!

09 HAVING Clause

The HAVING clause filters groups after aggregation — it is the equivalent of WHERE but for GROUP BY results.

WHERE vs HAVING

FeatureWHEREHAVING
FiltersIndividual rowsGrouped results
When appliedBefore GROUP BYAfter GROUP BY
Can use aggregates?NoYes
ExampleWHERE Salary > 50000HAVING AVG(Salary) > 50000
T-SQL
-- Departments with average salary above 50,000
SELECT
    Department,
    AVG(Salary) AS AvgSalary
FROM Employees
GROUP BY Department
HAVING AVG(Salary) > 50000;
💎
Performance Best Practice
Use WHERE first to reduce the number of rows before they reach GROUP BY. Only use HAVING for conditions that require aggregate values.

Constraints Reference

PRIMARY KEY

Uniquely identifies each row. Cannot be NULL. Only one per table.

FOREIGN KEY

References a primary key in another table. Maintains referential integrity.

UNIQUE

Prevents duplicate values in a column. Multiple allowed per table.

DEFAULT

Provides automatic values when no value is supplied on insert.

CHECK

Validates a condition before allowing data in. E.g. Salary > 0.

IDENTITY

Auto-increments integer values. Common for surrogate primary keys.

Production Caution for ALTER

11 DML Commands

Data Manipulation Language (DML) commands insert, update, and delete data within tables.

INSERT — Add Data

T-SQL
-- Single row insert (always specify column names)
INSERT INTO Employees (FirstName, LastName, Department, Salary)
VALUES ('John', 'Smith', 'HR', 50000);

-- Multi-row insert
INSERT INTO Employees (FirstName, LastName, Department, Salary)
VALUES
    ('Alice', 'Brown', 'Finance', 60000),
    ('David', 'Lee',   'Sales',   70000);

-- INSERT from SELECT (archive inactive records)
INSERT INTO EmployeeArchive
SELECT * FROM Employees
WHERE Status = 'Inactive';

UPDATE — Modify Existing Data

T-SQL
-- Targeted update with WHERE (safe)
UPDATE Employees
SET Salary = 55000
WHERE EmployeeID = 1;

-- Transaction safety pattern
BEGIN TRANSACTION;
    UPDATE Employees
    SET Salary = Salary + 5000
    WHERE Department = 'HR';
ROLLBACK; -- Change to COMMIT when verified

-- UPDATE with JOIN
UPDATE e
SET e.Salary = e.Salary + 5000
FROM Employees e
INNER JOIN Departments d ON e.Department = d.DepartmentName
WHERE d.Location = 'New York';

-- CASE-based conditional update
UPDATE Employees
SET Salary =
    CASE
        WHEN Department = 'HR'      THEN Salary + 1000
        WHEN Department = 'Finance' THEN Salary + 2000
        ELSE Salary
    END;

DELETE — Remove Rows

T-SQL
-- Delete specific row
DELETE FROM Employees WHERE EmployeeID = 5;

-- Delete with JOIN
DELETE o
FROM Orders o
INNER JOIN Customers c ON o.CustomerID = c.CustomerID
WHERE c.Country = 'USA';

-- Soft delete (preferred in production)
UPDATE Employees
SET Status = 'Inactive'
WHERE EmployeeID = 5;
💎
Soft Delete Strategy
Instead of physically deleting records, mark them as inactive using a Status column. This supports auditing, compliance, recovery, and historical analytics — critical in finance, HR, and healthcare systems.
MistakeRisk
Missing WHERE in UPDATEEvery row updated — potentially catastrophic
No transaction wrappingNo rollback path if something goes wrong
Wrong JOIN conditionUnexpected rows affected

12 LIKE Operator & Pattern Matching

T-SQL
-- Starts with A
WHERE CustomerName LIKE 'A%'

-- Ends with 'son'
WHERE CustomerName LIKE '%son'

-- Contains 'tech' anywhere
WHERE CustomerName LIKE '%tech%'

-- Single character wildcard (_)
WHERE CustomerName LIKE 'J_n'  -- Matches Jan, Jen, Jon

-- Character range
WHERE CustomerName LIKE '[A-C]%'

-- Escaping the % character itself
WHERE ProductCode LIKE '%\%%' ESCAPE '\'

Wildcard Summary

WildcardMeaningExample
%Zero or more characters'A%' → starts with A
_Exactly one character'J_n' → Jan, Jen, Jon
[abc]Any one character in set'[AK]%' → starts with A or K
[a-z]Any one character in range'[A-C]%' → starts with A, B, or C
[^abc]Any character NOT in set'[^AK]%' → not starting with A or K
⚠️
Leading Wildcard Kills Performance
LIKE '%A%' cannot use indexes (full table scan). LIKE 'A%' can use an index efficiently. Avoid leading wildcards on large tables.

13 NULL Handling

NULL means unknown — it is not zero, not empty string. It behaves uniquely in comparisons and aggregations.

T-SQL
-- IS NULL / IS NOT NULL
SELECT * FROM Employees WHERE ManagerID IS NULL;
SELECT * FROM Employees WHERE Email     IS NOT NULL;

-- ISNULL: T-SQL specific (2 arguments)
SELECT ISNULL(PhoneNumber, 'N/A') FROM Employees;

-- COALESCE: ANSI standard (multiple arguments)
SELECT COALESCE(PhoneNumber, MobileNumber, 'Not Available') FROM Employees;

NULL in Aggregate Functions

FunctionNULL Behavior
SUM(col)Ignores NULL values
AVG(col)Ignores NULL (affects denominator!)
COUNT(col)Ignores NULL rows
COUNT(*)Counts all rows including NULLs
MIN / MAXIgnores NULL values

COALESCE vs ISNULL

FeatureCOALESCEISNULL
StandardANSI SQLSQL Server only
ArgumentsMultiple (returns first non-NULL)Exactly two
FlexibilityMore flexibleSlightly faster

15 Aggregate Functions

T-SQL
-- Total salary payroll
SELECT SUM(Salary) AS TotalPayroll FROM Employees;

-- Average salary
SELECT AVG(Salary) AS AverageSalary FROM Employees;

-- Salary range
SELECT
    MIN(Salary) AS LowestSalary,
    MAX(Salary) AS HighestSalary
FROM Employees;

-- Row counts
SELECT COUNT(*) AS TotalRows          FROM Employees;
SELECT COUNT(Email) AS EmailsOnFile   FROM Employees; -- NULLs excluded
ℹ️
COUNT(*) vs COUNT(column)
COUNT(*) counts every row regardless of NULLs. COUNT(column) counts only rows where that column is not NULL. This difference matters greatly in analytics — always choose deliberately.

16 DISTINCT

The DISTINCT keyword removes duplicate rows from the result set.

T-SQL
-- Unique departments
SELECT DISTINCT Department FROM Employees;

-- Unique combinations of department + status
SELECT DISTINCT Department, Status FROM Employees;
💡
DISTINCT Applies to the Entire Row
SELECT DISTINCT Department, Status deduplicates by the combination of both columns, not just Department alone. Also note that DISTINCT may require internal sorting or hashing — use GROUP BY when you also need aggregation.

17 TOP Clause

The TOP clause limits the number of rows returned. Always pair it with ORDER BY for deterministic results.

T-SQL
-- Top 5 employees
SELECT TOP 5 * FROM Employees;

-- Top 5 highest paid (with ties)
SELECT TOP 5 WITH TIES *
FROM Employees
ORDER BY Salary DESC;

-- Pagination using OFFSET / FETCH NEXT
SELECT * FROM Employees
ORDER BY EmployeeID
OFFSET 10 ROWS
FETCH NEXT 10 ROWS ONLY;

-- Top 10 customers by spending
SELECT TOP 10
    CustomerID,
    SUM(TotalAmount) AS TotalSpent
FROM Orders
GROUP BY CustomerID
ORDER BY TotalSpent DESC;
Use CaseExample
Dashboard leaderboardTop 10 customers by revenue
HR reportingHighest-paid employees
Sales analysisTop-selling products
PaginationPage 2 of 10-row pages (OFFSET/FETCH)

19 Security & Production Safety

UPDATE & DELETE Safety Checklist

Checklist ItemImportance
✔ Always use WHERE clausePrevents accidental mass updates/deletes
✔ Test with SELECT firstVerify exactly which rows will be affected
✔ Wrap in a transactionEnables ROLLBACK if something looks wrong
✔ Backup critical tablesLast resort recovery
✔ Limit user permissionsPrinciple of least privilege

Production-Level Concerns

🔒 Locking

Large UPDATE or DELETE operations can lock tables and block other users. Run during off-peak hours or use batch deletes.

⏳ Blocking

Queries may wait indefinitely for locks held by other transactions. Monitor with sys.dm_exec_requests.

💀 Deadlocks

Two transactions waiting on each other's locks. SQL Server kills one. Design transactions to acquire locks in the same order.

22 Interview Questions & Answers

Q1
Difference between WHERE and HAVING?

WHERE filters individual rows before grouping. HAVING filters groups after aggregation and can reference aggregate functions like SUM or AVG. You cannot use aggregate functions in a WHERE clause.

Q2
Difference between DELETE and TRUNCATE?

DELETE removes rows one by one, supports WHERE, is fully logged, slower, and does not reset IDENTITY. TRUNCATE removes all rows at once, cannot use WHERE, is minimally logged, faster, and resets IDENTITY.

Q3
Why avoid SELECT *?

It causes unnecessary IO and network transfer, reduces performance, breaks applications when schema changes, reduces readability, and may expose sensitive columns inadvertently.

Q4
Difference between COUNT(*) and COUNT(column)?

COUNT(*) counts every row including those with NULLs. COUNT(column) only counts rows where the specified column is not NULL — important for nullable columns in analytics.

Q5
What is a PRIMARY KEY?

A constraint that uniquely identifies each row in a table. A primary key column cannot be NULL and must have unique values. Each table can have only one PRIMARY KEY (which may span multiple columns).

Q6
What is the logical query processing order?

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. This is why column aliases defined in SELECT cannot be referenced in WHERE, and why ORDER BY can use aliases.

Q7
What is a Foreign Key?

A constraint that enforces referential integrity by requiring a value in one table to match a value in the primary key of a related table. Prevents orphan records.

24 Advanced Pro Tips

Pro Tip 01
Always test UPDATE and DELETE with SELECT first.
T-SQL
-- Step 1: Verify the rows you'll affect
SELECT * FROM Employees WHERE Department = 'HR';

-- Step 2: Run the actual update
UPDATE Employees
SET Salary = Salary + 1000
WHERE Department = 'HR';
Pro Tip 02
Wrap critical modifications in transactions.
T-SQL
BEGIN TRANSACTION;

DELETE FROM Orders
WHERE OrderDate < '2020-01-01';

-- Verify row count before committing
ROLLBACK; -- change to COMMIT when satisfied
Pro Tip 03
Index strategically — not excessively.

Create indexes on columns used in WHERE, JOIN, and ORDER BY. Over-indexing slows INSERT, UPDATE, and DELETE. Review index usage with sys.dm_db_index_usage_stats.

Pro Tip 04
Never use functions on indexed columns in WHERE.

Wrapping a column in a function (e.g. YEAR(OrderDate)) prevents the query engine from using the index, causing full table scans. Use range comparisons instead.
